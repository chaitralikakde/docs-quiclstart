## Overview

Testing APIs that talk to Redis usually means either running Redis everywhere or spending time writing mocks that go stale. Keploy takes a different approach: it watches your running app, records every HTTP request and every Redis call your app makes, then plays it all back as a test suite. No test code to write, no mock libraries to set up.

This guide walks through a Go app built with [Gin](https://gin-gonic.com/) and Redis. The app has two endpoints: request an OTP and verify it.

---

<div id="how-it-works" style={{ scrollMarginTop: "5rem" }} />

## How it works

**During recording**, make API calls and Keploy saves every request and Redis interaction to disk.

**During testing**, Keploy replays those requests and fakes the Redis responses. No real Redis needed.

## Prerequisites

- [Go 1.21+](https://go.dev/dl/)
- [Docker Desktop](https://docs.docker.com/get-docker/) running

<Callout type="warning" title="macOS requirement">
  Docker Desktop 4.25.2 or later is required. Keploy uses eBPF to intercept network traffic, and older versions on macOS don't expose the kernel interfaces it needs.
</Callout>

## Quickstart

<StepCard id="step-1" step={1} title="Clone the repo">

<CodeBlock language="bash">{`git clone https://github.com/keploy/samples-go.git
cd samples-go/gin-redis
go mod download`}</CodeBlock>

</StepCard>

<StepCard id="step-2" step={2} title="Install Keploy">

Keploy is a CLI that wraps your app. No SDK or imports needed in your Go project.

<CodeBlock language="bash">{`curl --silent -O -L https://keploy.io/ent/install.sh && source install.sh`}</CodeBlock>

<CodeBlock language="bash">{`keploy --version`}</CodeBlock>

</StepCard>

<StepCard id="step-3" step={3} title="Create a Docker network and start Redis">

Containers are isolated by default. A shared network lets your app reach Redis using its container name (`myredis`) as a hostname.

<CodeBlock language="bash">{`docker network create keploy-network`}</CodeBlock>

<CodeBlock language="bash">{`docker run -p 6379:6379 -d --network keploy-network --name myredis redis`}</CodeBlock>

</StepCard>

<StepCard id="step-4" step={4} title="Update the Redis address and build the image">

Inside a Docker container, `localhost` refers to the container itself, not Redis. Change the address in `helpers/redis/redisConnect.go`:

<CodeBlock language="go" filename="helpers/redis/redisConnect.go">{`Addr: "myredis:6379",  // was localhost:6379`}</CodeBlock>

<CodeBlock language="bash">{`docker build -t gin-app:1.0 .`}</CodeBlock>

<Callout type="warning">
  Any code change requires a rebuild before it takes effect inside the container.
</Callout>

</StepCard>

<StepCard id="step-5" step={5} title="Record test cases">

This starts Keploy in record mode. Every HTTP request your app handles becomes a test. Every Redis call it makes becomes a mock.

<CodeBlock language="bash">{`keploy record -c "docker run -p 3001:3001 --network keploy-network --name ginRedisApp gin-app:1.0"`}</CodeBlock>

Wait for:

<CodeBlock>{`Keploy agent is ready to record test cases and mocks.`}</CodeBlock>

Keep this terminal open.

</StepCard>

<StepCard id="step-6" step={6} title="Make API calls">

Open a second terminal. These two requests become your test suite.

**Request an OTP:**

<CodeBlock language="bash">{`curl "localhost:3001/api/getVerificationCode?email=something@gmail.com&username=shivamsourav"`}</CodeBlock>

<CodeBlock language="json">{`{
  "status": "true",
  "message": "OTP Generated successfully",
  "otp": "5486"
}`}</CodeBlock>

**Verify the OTP:**

<CodeBlock language="bash">{`curl -X POST localhost:3001/api/verifyCode \
  --header 'Content-Type: application/json' \
  --data-raw '{"otp": 5486, "email": "something@gmail.com"}'`}</CodeBlock>

<CodeBlock language="json">{`{
  "status": "true",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "username": "shivamsourav",
  "message": "OTP authenticated successfully"
}`}</CodeBlock>

Press `Ctrl+C` in the first terminal to stop recording.

</StepCard>

<StepCard id="step-7" step={7} title="Run the tests">

<CodeBlock language="bash">{`keploy test -c "docker run -p 3001:3001 --rm --network keploy-network --name ginRedisApp gin-app:1.0" --delay 10`}</CodeBlock>

`--delay 10` gives the container 10 seconds to start before Keploy sends requests.

<Callout type="success" title="No Redis needed during tests">
  Keploy intercepts outbound Redis calls and returns the saved responses from `mocks.yml`. Tests run the same way locally, in CI, or anywhere else with no Redis required.
</Callout>

</StepCard>

## What Keploy captured

Keploy writes plain YAML files you can read, edit, and commit to git.

<img src="/test.png" alt="Generated test-1.yml file" style={{ borderRadius: "10px", border: "1px solid var(--border)", marginBottom: "1.25rem", width: "100%" }} />

`test-1.yml` holds the HTTP request and the expected response:

<CodeBlock language="yaml" filename="keploy/test-1.yml">{`version: api.keploy.io/v1beta2
kind: Http
name: test-1
spec:
  req:
    method: GET
    url: http://localhost:3001/api/getVerificationCode?email=something@gmail.com&username=shivamsourav
  resp:
    status_code: 200
    body: '{"status":"true","message":"OTP Generated successfully","otp":"5486"}'
  assertions:
    noise:
      - body.otp      # OTP changes every run, so Keploy skips asserting it
      - header.Date`}</CodeBlock>

The `noise` field tells Keploy which fields to skip when comparing. Anything non-deterministic (OTPs, timestamps, tokens) goes here.

<img src="/mocks.png" alt="Generated mocks.yml file" style={{ borderRadius: "10px", border: "1px solid var(--border)", marginTop: "1.25rem", marginBottom: "1.25rem", width: "100%" }} />

`mocks.yml` stores the raw Redis wire protocol. During tests, Keploy returns these exact bytes to your app so `go-redis` gets a response it can't tell apart from a real server:

<CodeBlock language="yaml" filename="keploy/mocks.yml">{`version: api.keploy.io/v1beta2
kind: Generic
name: mocks
spec:
  genericrequests:
    - origin: client
      message:
        - type: string
          data: "*2\r\n$3\r\nget\r\n$19\r\nsomething@gmail.com\r\n"
  genericresponses:
    - origin: server
      message:
        - type: string
          data: "$38\r\n{\"otp\":5486,\"username\":\"shivamsourav\"}\r\n"`}</CodeBlock>

## Troubleshooting

**`connection refused` on port 6379**

Update `Addr` to `myredis:6379` in `helpers/redis/redisConnect.go` and rebuild the image.

**`The container name "/myredis" is already in use`**

<CodeBlock language="bash">{`docker rm -f myredis`}</CodeBlock>

**`sh: networkName: No such file or directory`**

Replace `<networkName>` with the actual name, e.g. `keploy-network`.

**Keploy agent did not become ready in time**

<CodeBlock language="bash">{`docker rm -f ginRedisApp keploy-v3`}</CodeBlock>
